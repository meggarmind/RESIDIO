
## Security Contact Active Status Not Respecting Access Code Expiration
**Added**: 2025-12-19  
**Type**: Bug Fix  
**Module**: Security Contact  
**Priority**: High  
**Status**: Prompt request generated

Security contacts showing as Active in /security/contacts list even when their access codes have expired. Creates security concern as staff may assume contacts have valid access when they don't. Status in list view contradicts the expired status shown in detail view.

**Dependencies**: None
